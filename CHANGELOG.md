# Changelog

## Unreleased

### Added

- Initial standalone edgenet stack, ported from lumen-bootstrap.
- `docker-compose.yml` with the Lumen edgenet node (26657, 1317, 9090) and anvil forks for Base (8545) and Somnia (8546), all under one `edgenet` profile and one shared network.
- `Dockerfile.anvil`, a single shared image (foundry, curl, jq) reused by every anvil service, entrypoint `scripts/anvil-fork.sh`.
- Per chain fork height (`FORK_BLOCK`), sourced from `<NAME>_FORK_BLOCK` in `.env`. Empty or unset forks the upstream chain's current tip; a positive integer pins that height; anything else is fatal.
- `Makefile` with `make edgenet` as the single entrypoint plus per service start, stop and logs targets.
- `make build-force` and `make rebuild` Makefile targets. `build-force` builds all images with `--no-cache --pull`; `rebuild` runs `build-force` and then brings the stack up detached. Reach for `build-force` when a cached `RUN curl` layer is serving a stale artifact that was re-uploaded to the blob store under the same URL, since Docker keys that layer on the command text, not the response body. `--pull` also refreshes the base images, which matters because `foundry:stable` is a moving tag.
- `.env.example` documenting all configuration.
- `FUNDED_ACCOUNTS`, a comma separated list of extra bech32 addresses funded on the testnet in addition to the validator's own account, which is always funded. Ships in `.env.example` with the address that was previously hardcoded in `scripts/edgenet.sh`; leave it empty to fund only the validator.
- `README.md` covering prerequisites, quick start, services, configuration, adding an EVM chain and troubleshooting.
- `DEVELOPER.md` covering the architecture, the anvil fork entrypoint in `scripts/anvil-fork.sh`, the node boot sequence, the self-test, image build details and a known gaps backlog.

### Changed

- `scripts/edgenet.sh` now persists the chain across restarts instead of rebuilding it from the snapshot every boot. It checks for a sentinel file, `$CHAIN_HOME/initialized`: if present, it skips setup entirely (no clearing, no init, no snapshot fetch, no `in-place-testnet`) and execs `$BINARY start` against the existing chain home; if absent, it runs the full build as before and writes the sentinel immediately before invoking `in-place-testnet`. Since `CHAIN_HOME` is the bind mounted `.config/${CHAIN_ID}_edgenet/`, the sentinel survives restarts, so `restart: on-failure` no longer discards the chain on every crash. Three consequences worth knowing: changing `SNAPSHOT_URL` has no effect on an already-initialized container, since the resume path never reaches the snapshot code, `make clean` (or deleting the sentinel) is required to pick up a new one; the sentinel is written before `in-place-testnet` because that command never returns, so a conversion that dies partway leaves the sentinel over a half-converted home and the next restart runs `start` against it, recovery is `make clean`; and a crash from an unrelated cause, for example a corrupted database after a hard kill, now restarts into `start` against that same home and can crash-loop instead of self-healing, recovery is again `make clean`.
- **Breaking:** `SNAPSHOT_URL` is now a snapshot metadata JSON endpoint, not a direct archive URL. `scripts/edgenet.sh` reads `.url` and `.height` from it, downloads the archive from `.url`, and caches it by height. The old form, a direct `/download` archive URL, is now rejected; anyone with an existing `.env` must edit it to point at a metadata endpoint such as `https://snapshot.lumen.euclidprotocol.com/api/snapshots/latest` or a pinned `https://snapshot.lumen.euclidprotocol.com/api/snapshots/<height>`.
- **Breaking:** anvil fork block selection is now a plain pin. `BASE_FORK_BLOCK` / `SOMNIA_FORK_BLOCK` in `.env` map to a generic `FORK_BLOCK` per service in `docker-compose.yml`. Empty or unset forks the upstream chain's current tip; a positive integer pins that height; a non integer value is fatal. Anvil forks are no longer automatically time aligned with the Lumen snapshot; keeping a fork height consistent with the chosen snapshot is now a manual step the operator has to do themselves.
- The snapshot cache filename changed from `snapshot.tar.lz4` to `snapshot-<height>.tar.lz4`, keyed by the height reported in the snapshot metadata. `cache/` now grows by one archive per distinct height requested rather than being overwritten in place. Existing `cache/` directories may still hold an orphaned `snapshot.tar.lz4` from the old naming scheme; it is safe to delete by hand.
- `scripts/anvil-fork-test.sh` no longer exercises a fork block resolver against a simulated chain; it now exercises `FORK_BLOCK`'s tip, pinned and malformed cases end to end against a stubbed `anvil` binary, plus the shared quoted value guards in `scripts/anvil-fork.sh` and `scripts/edgenet.sh`.
- **Breaking:** `scripts/edgenet.sh` now invokes `in-place-testnet` with `--validator-operator`, `--validator-pubkey`, `--validator-privkey`, `--accounts-to-fund`, `--cosmwasm-admin` and `--coins-to-fund` instead of the old positional `in-place-testnet $CHAIN_ID $VALIDATOR_ADDRESS` form, matching how `EuclidProtocol/vsld` drives the same subcommand. All four validator values (`VAL_ACCOUNT`, `VAL_OPERATOR`, the consensus pubkey, the consensus privkey) are derived inside the container at runtime, via `keys show` on the recovered mnemonic and by reading `$CHAIN_HOME/config/priv_validator_key.json` with `jq`; none of them come from `.env`, and no key material is stored there. The consensus key is read only after the snapshot archive has been extracted, not right after `init`, because the snapshot's own `config/` can overwrite the key `init` generated; reading it earlier would hand `in-place-testnet` a consensus key that no longer matches what ends up on disk. The script now dies before `in-place-testnet` if `priv_validator_key.json` is missing, is not valid JSON, or has an empty key field, instead of passing an empty `--validator-privkey`.
- `--coins-to-fund` now lists `$DENOM` before `$STAKE_DENOM` (previously the reverse). Cosmos expects coin strings in sorted denom order, so this is a correctness fix, not a cosmetic reorder.

### Removed

- The fork block resolver in `scripts/anvil-fork.sh` that fetched snapshot metadata and searched the upstream chain for the block whose timestamp matched the snapshot's chain time. `SNAPSHOT_API_URL` and `BLOCK_TIME_MS` no longer exist anywhere in the stack.

### Fixed

- `Dockerfile.edgenet` now installs `lz4` and `bash`. The original image in lumen-bootstrap was missing both while `edgenet.sh` uses a bash shebang and calls `lz4 -dc` to extract the snapshot.
- `README.md` no longer claims `CHAIN_NAME` drives snapshot lookups. It is used for logging only.
- `scripts/edgenet.sh` no longer runs `rm -rf $CHAIN_HOME/` to clear the chain home on boot. `docker-compose.yml` bind mounts a host directory at that exact path, and a live mountpoint cannot be unlinked from inside the container holding it, so the old `rm -rf` failed with `rm: can't remove '/lumend/.lumend': Resource busy` and killed the entrypoint on every boot via `set -e`. The script now clears the directory's contents instead of removing the directory itself.
- `Dockerfile.edgenet`'s downloads of the chain binary and `genesis.json` now use `curl -fL` with retries, plus a post-download check (ELF magic bytes for the binary, `jq empty` for `genesis.json`). Neither download previously used `-f`, so an HTTP error such as a 404 was written to disk as if it were the artifact and the image built green; the binary became a 15-byte text file reading `Blob not found`, and `genesis.json` would have been whatever error page the blob store returned. The build now fails with an error naming the URL it could not fetch or validate.
- `Makefile` no longer uses `-include .env` plus a bare `export`. Make parses `.env` as makefile syntax, so quoted values from `.env.example` kept their literal quotes, and `export` pushed those quoted values into the environment that `docker compose` inherits, which outranks Compose's own `.env` parsing. This poisoned `BINARY` (baked into `ENV HOME` at image build time, producing `rm: can't remove '/"lumend"/."lumend"': Resource busy` in the `edgenet` container) and other quoted anvil variables (producing curl argument parsing errors in the anvil containers, back when the anvil entrypoint still shelled out to curl for snapshot metadata). Compose now loads `.env` natively, `.env.example` values are unquoted, and both entrypoints reject any value containing a quote character.
